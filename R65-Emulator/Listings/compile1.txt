  24 (    4) This is a Pascal derivative optimized for 8-bit
  25 (    4) microprocessors (integer type is 16 bit) with
  26 (    4) additional features (mem) to interact directly
  27 (    4) with the microprocessor hardware. Only one
  28 (    4) dimensional arrays and no records or user
  29 (    4) defined types. Floating point numbers (real)
  30 (    4) and file io to floppy disks are supported.
  31 (    4) 
  32 (    4) Precompiled libraries are merged in the loader.
  33 (    4) The table of reserved words and the library
  34 (    4) tables are loaded from the same drive as
  35 (    4) the compiler.
  36 (    4) 
  37 (    4) The output of the program is a loader file for
  38 (    4) the Pascal loader (compile2).
  39 (    4) 
  40 (    4) usage:
  41 (    4)  compile1 name[.cy[,drv]] [xxx]
  42 (    4)   where x:       l,p: no hard copy print
  43 (    4)                  i,r: index bound checking
  44 (    4)                  n: no loader file
  45 (    4)   [] means not required
  46 (    4) 
  47 (    4) The compiler uses 2 fixed memory areas to store
  48 (    4) the table of reserved words and the table of
  49 (    4) idents. The top of the Pascal stack (endstk) is
  50 (    4) adjusted accordingly. The reason for this hack
  51 (    4) is speed and convenience. A normal Pascal array
  52 (    4) of chars would store the chars as 16 bit
  53 (    4) numers and would therefore require twice the
  54 (    4) space. An array of packed chars would require
  55 (    4) more coding and slow the scanner module of the
  56 (    4) compiler down.                                }
